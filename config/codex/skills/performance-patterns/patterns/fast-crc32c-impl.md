# Fast CRC32C — implementation guide

## Library files

Three C files live in `library/`.  The agent picks which to include based on
the project's target hardware and portability requirements.

| File | Requires | Typical speed | When to use |
|------|----------|--------------|-------------|
| `crc32c_avx512_vpclmulqdq.c` | AVX-512F + VPCLMULQDQ (Ice Lake+, AMD Zen 4+) | 64–97 GB/s | Always include when targeting modern x86 |
| `crc32c_sse42.c` | SSE4.2 + PCLMULQDQ (any x86 from ~2010+) | 15–25 GB/s | Include as the intermediate fallback |
| `crc32c_portable.c` | None — pure C99 | 1–3 GB/s | Include for portability or non-x86 builds |

The AVX-512 file (`crc32c_avx512_vpclmulqdq.c`) was generated by the
[corsix fast-crc32](https://github.com/corsix/fast-crc32/) tool and is MIT or
zlib licensed.  Its file header must be preserved verbatim.  The SSE4.2 and
portable files are Intel originals (MIT license).

## Dispatch wrapper

Add this to a `.c` file in the project (e.g. `crc32c.c`).  Include whichever
`library/crc32c_*.c` files you have chosen to ship.

```c
/* CRC32C dispatch wrapper.
 * (C) 2026 Intel Corporation, MIT license
 * AVX-512 implementation from https://github.com/corsix/fast-crc32/
 * (MIT or zlib licensed) — see library/crc32c_avx512_vpclmulqdq.c
 */

#include <stddef.h>
#include <stdint.h>

/* Declarations of the implementations provided in library/. */
extern uint32_t crc32c_avx512_impl(uint32_t crc, const char *buf, size_t len);
extern uint32_t crc32c_sse42_impl (uint32_t crc, const char *buf, size_t len);
extern uint32_t crc32c_portable_impl(uint32_t crc, const char *buf, size_t len);

/* Remove declarations for implementations you are not linking. */

typedef uint32_t (*crc32c_fn)(uint32_t, const char *, size_t);

static crc32c_fn resolve_crc32c(void) {
#if defined(__x86_64__) || defined(_M_X64)
#  if defined(__GNUC__) || defined(__clang__)
    if (__builtin_cpu_supports("avx512f") &&
        __builtin_cpu_supports("vpclmulqdq"))
        return crc32c_avx512_impl;   /* remove if not linking this file */
    if (__builtin_cpu_supports("sse4.2"))
        return crc32c_sse42_impl;    /* remove if not linking this file */
#  endif
#endif
    return crc32c_portable_impl;
}

/* Public API — call this from application code. */
uint32_t crc32c(uint32_t crc, const void *buf, size_t len) {
    static crc32c_fn impl = NULL;
    if (__builtin_expect(impl == NULL, 0))
        impl = resolve_crc32c();
    return impl(crc, (const char *)buf, len);
}
```

### Adapting to an existing API

If the codebase already has a `crc32c()` function with a different signature,
rename the entry points above to match.  The typical variants are:

| Existing signature | Notes |
|---|---|
| `uint32_t crc32c(const void *buf, size_t len)` | Pass `crc = 0` to the impl |
| `uint32_t crc32c(uint32_t seed, const uint8_t *buf, size_t len)` | Cast buf to `const char *` |
| `uint64_t crc32c_hash(const void *data, size_t len, uint64_t seed)` | Narrow seed to `uint32_t`, return zero-extended |

## Compile flags

Each file needs specific compiler flags. The easiest approach is separate
compilation units:

```makefile
# Add to your build system
crc32c_avx512_vpclmulqdq.o: crc32c_avx512_vpclmulqdq.c
	$(CC) $(CFLAGS) -mavx512f -mavx512vl -mvpclmulqdq -O2 -c $< -o $@

crc32c_sse42.o: crc32c_sse42.c
	$(CC) $(CFLAGS) -msse4.2 -mpclmul -O2 -c $< -o $@

crc32c_portable.o: crc32c_portable.c
	$(CC) $(CFLAGS) -O2 -c $< -o $@
```

For CMake, use `target_compile_options` on the specific source files rather
than the whole target.

## Custom tuning with the code generator

The `v3s1_s3` algorithm in the shipped AVX-512 file is optimised for Sapphire
Rapids and performs well on Ice Lake too.  For a different microarchitecture,
regenerate the file with the [corsix generator](https://github.com/corsix/fast-crc32/):

```bash
git clone https://github.com/corsix/fast-crc32
cd fast-crc32
make generate
# Ice Lake / AMD Zen 4 (best for those):
./generate -i avx512_vpclmulqdq -p crc32c -a v4s5x3
# AMD Genoa:
./generate -i avx512_vpclmulqdq -p crc32c -a v3s2x4
# No AVX-512, SSE4.2 only, best general tuning:
./generate -i sse -p crc32c -a v8s3x3
```

Rename the generated `crc32_impl` function to `crc32c_avx512_impl` (or
`crc32c_sse42_impl`) to match the dispatch wrapper above.

The terse algorithm string format:
- `v4` — 4 vector (ZMM) accumulators
- `s5` — 5 scalar (`_mm_crc32_u64`) accumulators
- `x3` — 3× loads per accumulator per iteration
- `_s3` — fallback to 3-scalar mode for remaining bytes
